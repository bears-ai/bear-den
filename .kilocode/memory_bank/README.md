# Memory Bank

This directory serves as the project's memory bank - a centralized location for storing important project information, decisions, and documentation.

## Contents

- [`project_brief.md`](project_brief.md) - The main project brief document containing goals, scope, requirements, and other key project information

## Purpose

The memory bank helps maintain project context and knowledge by:
- Documenting key decisions and rationale
- Storing project requirements and specifications
- Maintaining a single source of truth for project information
- Facilitating onboarding and knowledge transfer

## Usage

Feel free to add additional documents to this directory as needed, such as:
- Architecture decisions (ADRs)
- Meeting notes
- Technical specifications
- Design documents
- Research findings

Keep documents organized and up-to-date to maximize their value for the team.

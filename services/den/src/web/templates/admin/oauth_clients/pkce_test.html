About PKCE Testing

This tool allows you to test and validate PKCE (Proof Key for Code Exchange) flows for your OAuth client. PKCE provides enhanced security for public clients by preventing authorization code interception attacks.

How to Use:

  1. Generate Sample Values: Use the provided sample code verifier and challenge for testing
  2. Test Validation: Enter your own values and click "Test PKCE Validation"
  3. Review Results: See if the validation succeeds or fails with detailed feedback

Supported Challenge Methods:

  • S256: SHA256 hash of the code verifier (recommended for production)
  • plain: Code challenge equals code verifier (less secure, for testing only)

How PKCE Works:

  1. Client generates: A random code verifier (43-128 chars)
  2. Client creates challenge: SHA256 hash of verifier (base64url encoded) for S256 method
  3. Authorization request: Includes code challenge and method
  4. Token request: Includes original code verifier
  5. Server validates: Recomputes challenge from verifier and compares
